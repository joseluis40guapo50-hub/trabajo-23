Modelo Híbrido de Metodologías de Desarrollo
Cascada + Scrum

En esta práctica se implementa un modelo híbrido que combina las metodologías
Cascada y Scrum, aplicado al proyecto simulado “Gestión de Tareas”.
El objetivo es aprovechar las fortalezas de ambas metodologías para organizar
el trabajo de manera clara y flexible.

---

1. Justificación de la elección de metodologías

Metodología Cascada
Se eligió Cascada porque:
- Permite una planificación clara desde el inicio
- Define de manera ordenada las fases del proyecto
- Facilita la documentación en proyectos académicos

Metodología Scrum
Se eligió Scrum porque:
- Aporta flexibilidad durante el desarrollo
- Permite dividir el trabajo en iteraciones cortas
- Facilita el seguimiento del progreso mediante entregables frecuentes

---

2. Aporte de cada metodología al modelo híbrido

Metodología Aporte principal 
|------------|------------------|
| Cascada | Organización inicial, documentación y estructura |
| Scrum | Iteraciones, adaptación a cambios y control del progreso |


3. Aplicación del modelo híbrido en el proyecto

Fase 1: Planificación y Análisis (Cascada)
- Definición del objetivo del proyecto
- Identificación de funcionalidades principales
- Documentación inicial del alcance

Entregables:
- Documento de planificación
- Alcance del proyecto definido


Fase 2: Diseño General (Cascada)
- Diseño de la estructura del proyecto
- Definición de componentes y archivos

Entregables:
- Documento de diseño
- Estructura del repositorio

---

Fase 3: Desarrollo Iterativo (Scrum)
El desarrollo se divide en **sprints** de 1 semana:

Sprint 1
- Implementación básica del proyecto
- Creación de archivos principales

Sprint 2
- Mejora de funcionalidades
- Organización del código y documentación

Entregables por sprint:
- Incremento del proyecto
- Commits claros y estructurados


 Fase 4: Pruebas y Cierre (Scrum + Cascada)
- Revisión general del proyecto
- Corrección de errores simples
- Documentación final

Entregables:
- Documento de pruebas
- Versión final del proyecto simulado

4. Tiempos estimados

Fase  Duración 
|----|----|
| Planificación y análisis | 1 semana |
| Diseño | 1 semana |
| Desarrollo (2 sprints) | 2 semanas |
| Pruebas y cierre | 1 semana |

Duración total estimada: 5 semanas

---

5. Conclusión

El uso de un modelo híbrido Cascada + Scrum permitió combinar la
estructura y planificación detallada de Cascada con la flexibilidad
y mejora continua de Scrum. Este enfoque facilitó la organización
del proyecto y el control del progreso mediante Git y GitHub,
incluso en un entorno de desarrollo simulado.
